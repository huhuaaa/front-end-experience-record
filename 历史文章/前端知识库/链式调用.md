# 链式调用

## setTimeout/setInterval

运行机制详解
setTimeout 和 setInterval 的执行过程并非“倒计时结束立即执行”，而是分为两个阶段：‌注册计时‌和‌入队等待‌。

第一阶段：注册计时（Web API层）
当代码执行到 setTimeout(fn, delay) 时：

JS 引擎立即将该定时器注册到浏览器的 ‌Web API 模块‌（通常是定时器线程）。
Web API 模块开始计时。
‌关键点‌：此时 fn ‌不会‌被阻塞等待，JS 主线程会继续向下执行后续同步代码。
第二阶段：入队等待（任务队列）
当 Web API 模块计时结束（达到 delay 毫秒后）：

它会将回调函数 fn 放入 ‌宏任务队列（MacroTask Queue / Callback Queue）‌。
‌注意‌：放入队列不代表立即执行。如果此时主线程正在执行其他任务（如同步代码、其他回调），fn 必须在队列中排队等待。
第三阶段：事件循环调度（Event Loop）
事件循环不断监控调用栈和任务队列：

当‌调用栈（Call Stack）为空‌（即当前所有同步代码执行完毕）。
事件循环从‌宏任务队列‌中取出第一个任务（即我们的定时器回调）。
将该任务推入调用栈执行。

## 考试题

实现一个CodingMan，可以按照以下方式调用:

题目描述
​
CodingMan(“Hank”)​
输出:​
Hi! This is Hank!​

CodingMan(“Hank”).eat(“dinner”).eat(“supper”)​
输出​
Hi This is Hank!​
Eat dinner~​
Eat supper~​

CodingMan(“Hank”).sleep(10).eat(“dinner”)​
输出​
Hi! This is Hank!​
//等待10秒..​
Wake up after 10​
Eat dinner~​

CodingMan(“Hank”).sleepFirst(5).eat(“supper”)​
输出​
//等待5秒​
Wake up after 5​
Hi This is Hank!​
Eat supper​
以此类推。​​
​​​
评价
考察点：面向对象封装、链式调用、异步/同步队列处理
作答情况：链式调用的实现不了解，缺少封装性，没有想到比较好的任务处理方式

```JavaScript
class CodingManObject {
  constructor(name) {
    this.name = name;
    this.queue = []; // 任务队列

    // 默认任务：打招呼
    const sayHi = () => {
      console.log(`Hi! This is ${this.name}!`);
      this.next(); // 执行完当前任务，继续下一个
    };

    // 将打招呼加入队列
    this.queue.push(sayHi);

    // 使用 setTimeout 确保在构造函数执行完毕、后续链式调用添加完任务后，才开始执行队列
    // 这样可以让 sleepFirst 有机会通过 unshift 插队到 sayHi 之前
    // setTimeout函数执行会在当前同步方法执行完成之后，再执行；其原理是因为setTimeout本质上是将内部函数扔到一个宏任务队列里面去等待同步代码执行完成后调用。
    setTimeout(() => {
      this.next();
    }, 0);
  }

  // 从队列中取出第一个任务并执行
  next() {
    // 先进先出
    const task = this.queue.shift();
    if (typeof task === 'function') {
      task();
    }
  }

  eat(food) {
    const task = () => {
      console.log(`Eat ${food}~`);
      this.next();
    };
    this.queue.push(task);
    return this; // 返回 this 实现链式调用
  }

  sleep(time, isFirst = false) {
    const task = () => {
      setTimeout(() => {
        console.log(`Wake up after ${time}`);
        this.next();
      }, time * 1000);
    };
    // 关键：unshift 将任务插入到队列最前面
    // 因为构造函数中 setTimeout 是宏任务，而链式调用是同步执行的
    // 所以当代码执行到 sleepFirst 时，sayHi 还在队列里
    // unshift 后，队列变为 [sleepFirstTask, sayHiTask, ...]
    isFirst ? this.queue.unshift(task) : this.queue.push(task);
    return this;
  }

  sleepFirst(time) {
    return this.sleep(time, true);
  }
}

// 工厂函数，方便直接调用 CodingMan('Hank') 而不是 new CodingMan('Hank')
function CodingMan(name) {
  return new CodingManObject(name);
}

CodingMan("Hank").eat("dinner").eat("supper");
CodingMan("Hank").sleep(10).eat("dinner");
CodingMan("Hank").sleepFirst(5).eat("supper");
```
