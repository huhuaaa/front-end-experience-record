# 一、先分清三个概念（必考）

1. **Layout（重排 / Reflow）**
   - 元素**几何尺寸、位置**发生变化
   - 浏览器重新计算布局
   - **性能开销最大**

2. **Paint（重绘 / Repaint）**
   - 只改**颜色、背景、透明度**等外观
   - 不影响布局
   - 开销比重排小很多

3. **Composite（合成）**
   - 只改变 `transform`、`opacity`
   - 直接用 GPU 合成
   - **开销最小，几乎不卡**

# 二、什么会触发重排？（高频）

只要影响**布局/尺寸/位置**都会触发：

- 宽高：`width` `height` `padding` `margin`
- 位置：`top` `left` `right` `bottom`
- 显示隐藏：`display: none`
- 增删 DOM
- 获取布局类属性：
  `offsetWidth` `clientHeight` `getBoundingClientRect` 等
- 浏览器窗口 resize

> 口诀：**一动布局必重排，一拿尺寸也重排**

---

# 三、什么只会触发重绘？

- `color`
- `background-color`
- `visibility`
- `box-shadow`
- 不影响布局的样式

---

# 四、什么不触发重排重绘，只走合成？（超级加分）

只有两个：

1. **transform**
2. **opacity**

股票价格闪烁、盘口高亮、买入卖出动画，**优先用这两个**，性能最好。

---

# 五、如何减少重排重绘？（面试标准答案）

1. **批量修改样式**
   不要一条一条改 style，统一加 class。

2. **离线操作 DOM**
   `display: none` 改完再显示
   或使用 DocumentFragment

3. **避免频繁读取布局属性**
   不要一边读 `offsetWidth` 一边改样式，会导致**强制同步布局**。

4. **使用 transform / opacity 做动画**
   不走重排重绘，GPU 加速。

5. **开启 will-change**
   告诉浏览器这个元素要频繁变，提前优化。

6. **避免 table 布局**
   很小的改动都会触发整个 table 重排。

---

# 六、金融/行情场景怎么用？（实战加分）

股票价格、盘口、买卖五档经常变：

- 不要频繁改 `top` / `height`
- 涨跌颜色变化没问题（重绘）
- 闪烁动画用 `transform: scale(1.05)` 或 `opacity`
- 大量更新时，先**隐藏面板 → 改数据 → 再显示**，只触发一次重排

---

# 七、极简背诵版（30 秒）

- **重排**：元素尺寸位置变化，开销最大。
- **重绘**：只改颜色样式，开销较小。
- **合成**：transform、opacity 性能最好。
- 优化思路：批量修改、用 class、避免读写布局、优先用 transform/opacity。
