# 一、HTTP 缓存两大类

1. **强缓存**  
2. **协商缓存**

一句话区分：

- **强缓存：直接用本地，不发请求**
- **协商缓存：发请求问服务器，能用再用本地**

---

# 二、强缓存（Strong Cache）

## 1. 规则

浏览器直接读本地缓存，**完全不发请求**，状态码 **200 OK (from disk cache)**。

## 2. 控制字段

### ① `Expires`（HTTP/1.0）

- 绝对时间
- 受本地时间影响，有缺陷

### ② `Cache-Control: max-age=xxx`（HTTP/1.1，现代主流）

- 相对时间，单位秒
- 优先级比 Expires 高

常用值：

- `max-age=31536000` 长期缓存（JS/CSS/图片带hash）
- `no-cache` **不走强缓存，走协商缓存**
- `no-store` **完全不缓存**（金融接口常用！）

---

# 三、协商缓存（Negotiation Cache）

## 1. 规则

发一次请求给服务器，服务器判断：

- 资源没变 → 返回 **304 Not Modified**，用本地
- 资源变了 → 返回 **200 + 新内容**

## 2. 两组配对字段

### 第一组：`Last-Modified` / `If-Modified-Since`

- 服务端返回：`Last-Modified: 时间`
- 下次请求带上：`If-Modified-Since: 时间`
- 服务端对比时间

### 第二组：`ETag` / `If-None-Match`（优先级更高）

- 服务端返回：`ETag: "xxx"`（文件指纹/哈希）
- 下次请求带上：`If-None-Match: "xxx"`
- 精准判断文件是否变化，比时间更可靠

---

# 四、执行流程（面试必说）

1. 先看是否命中**强缓存**（Cache-Control / Expires）
   - 命中 → 直接用，不发请求
2. 没命中 → 走**协商缓存**
   - 发请求带 `If-Modified-Since` / `If-None-Match`
   - 服务端返回 304 → 用缓存
   - 返回 200 → 用新内容

---

# 五、金融/股票项目怎么用？（高频考点）

1. **JS/CSS/静态图片**
   - 文件名带 hash → 设置 `max-age=31536000` 强缓存

2. **行情、交易接口**
   - 绝对不能强缓存！
   - 配置：

     ```Text
     Cache-Control: no-cache, no-store, max-age=0
     ```

   防止价格/持仓数据缓存旧的，导致**交易风险**。

3. **K线图、公告图片**
   - 可短期强缓存或协商缓存

---

# 六、极简背诵版（30 秒满分回答）

- **强缓存**：由 `Cache-Control: max-age` 控制，**不发请求**直接使用本地。
- **协商缓存**：通过 `Last-Modified/ETag` 对比，**发请求**，服务端返回 **304** 则使用缓存。
- 执行顺序：先强缓存，再协商缓存。
- 金融接口必须禁用缓存，防止价格、订单数据展示错误。
