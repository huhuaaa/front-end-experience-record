# 动态规划

## 一、满足这 3 个特征，优先想 DP

最优子结构
大问题的最优解，可以由子问题的最优解推出来。
比如：最短路径、最长递增子序列。
重叠子问题
暴力递归时，会反复计算一模一样的子问题。
DP 就是用数组 / 表把结果存下来，避免重复算。
无后效性
一旦确定了当前状态，未来的决策只跟当前状态有关，跟怎么走到这个状态的路径无关。

## 二、最典型适合 DP 的场景

1. 计数类问题
   走楼梯：n 阶楼梯，一次走 1/2 步，有多少种走法
   硬币组合：凑出金额有多少种方式
   字符串分割、路径条数（网格从左上到右下有几条路）
2. 最值类问题
   最长递增子序列 LIS
   最长公共子序列 LCS
   最大子数组和
   打家劫舍（不能相邻，求最大收益）
   背包问题（01 背包、完全背包）
3. 决策类问题
   买卖股票最佳时机
   编辑距离（最少修改几步让字符串相同）
   矩阵链乘法（最小计算量）
   扔鸡蛋问题
4. 划分 / 分割问题
   分割回文串最少切割次数
   数组分割成和相等的子集
   整数拆分（拆成若干数乘积最大）
5. 博弈类问题
   取石子游戏
   预测赢家（两人轮流取数，是否先手必胜）
   三、一眼识别 DP 的关键词
   题目里出现这些词，优先 DP：
   最多、最少、最大、最小
   多少种方案、多少种方法
   最长、最短
   子序列、子数组、子集
   不能相邻、不能重复选择
   从前 i 个里选、状态只和前一步有关
   四、不适合 DP 的场景
   问题没有重复子问题，递归一次算完不重复 → 贪心 / 递归更合适
   贪心能直接一步到位（如活动选择、哈夫曼编码）
   问题是排列组合全枚举，且状态无法压缩
   后效性很强，未来会影响过去状态

## 三、样例题

假设你正在爬楼梯，一共 n 阶台阶。
每次只能爬 1 阶 或 2 阶。
问：有多少种不同的方法可以爬到楼顶？

1是：1，共1种
2是：1 1，2，共2种
3是：1 1 1，1 2, 2 1 2，共3种
4是：1 1 1 1，1 2 1, 2 1 1，1 1 2，2 2，共5种

除了1、2两个台阶是固定的，每增加1阶梯，是前两个之和。
就是dp[n] = dp[n - 1] + dp[n - 2]

```TypeScript
/**
 * 爬楼梯 - 动态规划
 * @param n 台阶数
 * @returns 方法总数
 */
function climbStairs(n: number): number {
  // 边界
  if (n === 1) return 1;
  if (n === 2) return 2;

  // dp 数组：dp[i] 表示第 i 阶的方法数
  const dp: number[] = new Array(n + 1);

  dp[1] = 1;
  dp[2] = 2;

  // 递推
  for (let i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}

// 测试
console.log(climbStairs(5)); // 8
console.log(climbStairs(10)); // 89
```

```TypeScript
// 空间优化版本
/**
 * 爬楼梯 - 动态规划
 * @param n 台阶数
 * @returns 方法总数
 */
function climbStairs(n: number): number {
    // 边界
    if (n === 1) return 1;
    if (n === 2) return 2;

    // 因为只需要记住前一、二两阶数据即可
    let preCount = 2;
    let ppreCount = 1;
    let current;

    // 递推
    for (let i = 3; i <= n; i++) {
        if (i > 3) {
            ppreCount = preCount;
            preCount = current;
        }
        current = ppreCount + preCount;
    }

    return current;
}

// 测试
console.log(climbStairs(5)); // 8
console.log(climbStairs(10)); // 89
```

### 0-1 背包问题描述

有一个背包，最大承重为 capacity。
有 n 个物品，每个物品：
重量 weights[i]
价值 values[i]
每个物品只能选一次（0 不选 / 1 选）。
求在不超过背包容量的前提下，能装的最大价值。

解题思路

状态定义
dp[i][j]：前 i 个物品，背包容量为 j 时的最大价值。
状态转移
对第 i 个物品，有两种选择：
不选：dp[i][j] = dp[i-1][j]
选：dp[i][j] = dp[i-1]j - weights[i-1]] + values[i-1]

去最大值
dp[i][j] = max(
dp[i-1][j],
dp[i-1][j - w] + v
)

```TypeScript
/**
 * 0-1 背包 二维动态规划
 * @param weights 物品重量数组
 * @param values 物品价值数组
 * @param capacity 背包最大容量
 * @returns 最大价值
 */
function knapsack01(
  weights: number[],
  values: number[],
  capacity: number
): number {
  const n = weights.length;

  // dp[i][j] = 前i个物品，容量j时的最大价值；此处用n+1和capacity+1，是为了给0 0，占用第一个位置。
  const dp: number[][] = Array.from({ length: n + 1 }, () =>
    new Array(capacity + 1).fill(0)
  );

  for (let i = 1; i <= n; i++) {
    const w = weights[i - 1];
    const v = values[i - 1];
    for (let j = 1; j <= capacity; j++) {
      if (j < w) {
        // 仅当前一个就超标，那么肯定无法选择，只需要获取第i-1个物品的最大价值即可
        dp[i][j] = dp[i - 1][j];
      } else {
        /**
         * 放得下时，分两种情况：
         * 1. 放进去当前物品后，剩余负重量从i-1个物品中算出一个最大，叠加即是当前放入第i个物品的最大值。
         * 2. 跟不放进入第i个物品做比较，取大者就是最大值。
         */
        dp[i][j] = Math.max(dp[i - 1][j], dp[i - 1][j - w] + v);
      }
    }
  }

  return dp[n][capacity];
}

// 测试
const weights = [2, 3, 4, 5];
const values = [3, 4, 5, 6];
const capacity = 8;

console.log(knapsack01(weights, values, capacity)); // 10
```

```TypeScript
// 空间优化版本
function knapsack01Opt(
  weights: number[],
  values: number[],
  capacity: number
): number {
  const dp = new Array(capacity + 1).fill(0);
  const n = weights.length;

  for (let i = 0; i < n; i++) {
    const w = weights[i];
    const v = values[i];

    // 关键：倒序遍历，避免重复选
    for (let j = capacity; j >= w; j--) {
      dp[j] = Math.max(dp[j], dp[j - w] + v);
    }
  }

  return dp[capacity];
}

// 测试
const weights = [2, 3, 4, 5];
const values = [3, 4, 5, 6];
const capacity = 8;
console.log(knapsack01Opt(weights, values, capacity)); // 10
```
